What is Subnetting?

Subnetting is the process of dividing a large IP network into smaller, more manageable sub-networks (subnets). This practice improves network performance, enhances security, and reduces broadcast traffic by isolating network segments.

Subnetting works by borrowing bits from the host portion of an IP address to create additional network bits. For example, a /24 network (255.255.255.0) can be subnetted into two /25 networks (255.255.255.128), each with 126 usable hosts.

Key benefits of subnetting include:

  • Improved performance — Reduced broadcast domains mean less network congestion
  • Better security — Isolated segments limit the spread of threats
  • Efficient IP usage — Allocate only the addresses you need
  • Simplified management — Logical grouping of devices by function or location

What is CIDR Notation?

CIDR (Classless Inter-Domain Routing) notation is a compact way to represent an IP address and its associated subnet mask. It is written as an IP address followed by a slash and the number of leading 1-bits in the subnet mask.

For example, 192.168.1.0/24 means the first 24 bits (3 bytes) represent the network portion, and the remaining 8 bits represent the host portion. This is equivalent to the subnet mask 255.255.255.0.

CIDR replaced the older classful addressing system (Class A, B, C) because it allows more flexible allocation of IP addresses. Common CIDR prefixes include:

  • /8 — 16,777,216 addresses (Class A equivalent)
  • /16 — 65,536 addresses (Class B equivalent)
  • /24 — 256 addresses (Class C equivalent, most common for LANs)
  • /30 — 4 addresses (common for point-to-point links)
  • /32 — Single host address

What is a Broadcast Address?

The broadcast address is the last address in a subnet. It is used to send data to all hosts on the network simultaneously. When a device sends a packet to the broadcast address, every device on that subnet receives it.

The broadcast address is calculated by setting all host bits to 1. For example, in the network 192.168.1.0/24, the broadcast address is 192.168.1.255.

Broadcast addresses cannot be assigned to individual hosts. They are reserved for network-wide communication such as ARP requests, DHCP discovery, and routing protocol updates.

What is a Network Address?

The network address (also called the network ID) is the first address in a subnet. It identifies the network itself and cannot be assigned to a host device.

The network address is calculated by performing a bitwise AND operation between the IP address and the subnet mask. For example, 192.168.1.10 with subnet mask 255.255.255.0 gives network address 192.168.1.0.

Network addresses are used in routing tables to determine where to send packets. Routers use the network address to make forwarding decisions, directing traffic to the correct destination network.

IPv4 Address Classes Explained

IPv4 addresses were originally divided into five classes (A through E) based on the first few bits of the address. While modern networking uses CIDR, understanding classes helps with legacy systems and certification exams.

How to Calculate Subnets

Calculating subnets manually involves several steps. Here is a practical guide using 192.168.1.10/24 as an example:

  1. Convert IP to binary: 192.168.1.10 = 11000000.10101000.00000001.00001010
  2. Determine subnet mask: /24 = 255.255.255.0 = 11111111.11111111.11111111.00000000
  3. Calculate network address: Perform bitwise AND between IP and mask → 192.168.1.0
  4. Calculate broadcast address: Set all host bits to 1 → 192.168.1.255
  5. First usable host: Network address + 1 → 192.168.1.1
  6. Last usable host: Broadcast address - 1 → 192.168.1.254
  7. Total hosts: 2^(32-24) = 256
  8. Usable hosts: 256 - 2 = 254
